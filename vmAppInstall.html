Artifacts - Application Installation on a VM

Description: This artifact shows how to install an application on an existing virtual machine running Debian Linux on Google Cloud Platform (GCP). It involves downloading both a required Java runtime and the intended application, Minecraft. Once downloaded, the target application is then setup and tested to verify it runs correctly via verification of a remote client login.
Related Class: Kent State University, IT-36339, Cloud and Virtualization Technologies, Spring 2025
Selection Rationale: This artifact was chosen because a significant amount of my personal time is spent working on code and game development. This evidences my ability to deploy a software package to be used in an IT environment and configure it at a low level for use.
Competencies Demonstrated: This artifact showcases my ability to take an application, gather its installation requirements, deploy it in live environment, and then verify that users can connect properly.
Expected Outcome: As a result of this artifact presentation, an application server running Minecraft code will be successfully installed and enable a client to properly connect and use its services.
Skills Obtained: Virtualization fundamentals, cloud administration, Linux command‑line administration, package installation and dependency management, application configuration, software deployment, configuration file editing, remote access configuration, service testing and validation, troubleshooting.

Step 1: Find the version of Java for your platform if not already installed on your system. To do this, go to https://jdk.java.net/archive and download the latest version of the Open JDK. In this case, we are running on an x64 based Debian Linux virtual machine, so we will need to download the Linux/X64 tar archive. Note: You can also use Oracle Java, but many businesses have switched over to Open JDK due to licence concerns.
Step 2: Now that we have found the correct download link for our platform, Java needs to be downloaded. To do this, Linux has a utility called wget that can directly download a remote file via the command line when it is passed the URI as an argument (wget URI). So, we now execute wget on our java tar archive to download it to the local filesystem.
Step 3: We now need to extract Java runtime files from the archive that we have downloaded. To do this on the specific file we downloaded, we execute tar -xzvf openjdk-25_linux-x64_bin.tar.gz, which does this with arguments that specify to extract (x), gnu unzip (z), output log info verbosely (v), and use the specified file (f) argument.
Step 4: Here, we are just verifying that Java is installed and runs successfully. To do this, we issue the command cd ~/jdk-25/bin, then execute ./java as a command from within the installed java binary executable directory.
Step 5: Now that Java is installed, we can download our application, Minecraft. To do this, we go to http://minecraft.net/en-us/download/server, which will display the latest server Java jar file URI along with the command line required to execute and run the server, java -Xmx1024M -Xms1024M -jar minecraft_server.1.21.10.jar nogui. Once this is viewable, copy the URI to the Minecraft jar file as listed on the web page into our copy buffer, then transition back over to our ssh command window.
Step 6: Just like was done previously to download Java to our server, we issue a wget URI for the URI of the Minecraft server jar that is currently stored in our paste buffer.
Step 7: Now that we have the Minecraft server jar file downloaded, we can execute it to start up the application. Note: When downloading the server jar file, it was named server.jar, so we need to modify our java command line to use this jar file name to java -Xmx1024M -Xms1024M -jar server.jar nogui.
Step 8: After starting the application in the previous step, if aborts with the error shown here. This is because when starting the Minecraft server application for the first time, it is supposed to automatically fail. This is because you are required to accept the end user license agreement (EULA).
Step 9: As an output of this forced error, a file is created on the local filesystem where the server.jar is run from with the name eula.txt. Edit the file to change eula=false to instead say eula=true. Then save it.
Step 10: Once this is completed and the eula.txt file is saved, you can then fully start the Minecraft server application. Again, by running java -Xmx1024M -Xms1024M -jar server.jar nogui. Once fully up, you will see the value Done output to the console log.
Step 11: On any desktop PC with internet access and the Minecraft client installed, you can now startup the Minecraft client application. This image shows the initial client startup screen that displays while it loads.
Step 12: Once the client is up and fully loaded, select Add Server on the client, enter the public IP address of our new Minecraft application server, then click done to connect. (The public IP configured on Google Cloud Platform,in this case, was 34.153.17.166)
Step 13: Success! The client has connected to the remote Minecraft application server. This also shows on the console log with a user connect statement showing: EnderReaper64 joined the game.
