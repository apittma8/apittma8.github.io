Artifacts - C# Credit Card Validator (Elysia API, Visual Studio, & Copilot)

Description: This artifact demonstrates the development of a C# console application that interacts with the Elysia Tools Credit Card Validator API. I established the project framework, including the folder structure, model classes, and service class, before using GitHub Copilot to assist with generating the initial code. Although Copilot provided useful and mostly working code, it did not always return the requested response format, even after several prompting attempts. After manual changes, the code then fully behaved as disred. After this, I reviewed and refined it through the web version of Copilot and manually removed unnecessary validations and formatting to align the implementation with production best practices. The resulting codebase is clean, maintainable, and correctly processes the API’s returned data.

Step 1: Create a New Project. To do this, I opened Visual Studio, created a new project as a Console App (.NET framework), then named it cardValidator.
Step 2: Setup the base framework. Create the Services and Models directories, and associated cs files under each. Program.cs is created in the root folder. The Models folder holds the request and response objects (ccValidateRequest.cs and ccValidateResponse.cs) that are used when talking to the API and the Services folder stores the code (ccValidateService.cs) that sends the credit card number to the API and returns the results.
Step 3: Now the code is developed which creates the service, calls the API, and returns the results.
