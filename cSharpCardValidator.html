Artifacts - C# Credit Card Validator (Elysia API, Visual Studio, & Copilot)


Description: This is a C# console application that interacts with the Elysia Tools Credit Card Validator API. I created the project framework, including the folder structure, model classes, and service class, before using GitHub Copilot to assist with generating the initial code. Although Copilot provided useful and mostly working code, it did not always return the requested response format, even after several prompting attempts. After manual changes, the code then fully behaved as desired. After this, I reviewed and refined it through the web version of Copilot and manually removed unnecessary validations and formatting to align the implementation with production best practices. The resulting codebase is clean, maintainable, and correctly processes the API’s returned data.
Related Class: Kent State University, IT-11002, Visual Basic / Visual Studio, Fall 2021
Other Training: C# development using Unity/Mono and Terraria/XNA in personal projects
Selection Rationale: This artifact was selected because it demonstrates my ability to design and implement an API-based application using C# and .NET within a development environment. It also highlights my ability to use AI tools while retaining responsibility for architectural decisions, code accuracy, and quality.
Competencies Demonstrated: This is demonstrated in consuming REST APIs, organizing code into logical components, and working with JSON serialization and asynchronous programming patterns. It also reflects my ability to critically evaluate AI‑generated code, identify mismatches with actual API requirements, and make necessary corrections to ensure everything works properly.
Expected Outcome: The completed application validates a user‑provided credit card number by sending it to the Elysia Tools Credit Card Validator API, processing the returned JSON, and displaying the results in a clear, readable format.
Skills Obtained: C# development, .NET console application design, REST API use, JSON input and output processing, asynchronous programming, code organization, AI‑assisted development, code review and editing, error handling, software debugging and testing

Step 1: Create a New Project. To do this, I opened Visual Studio, created a new project as a Console App (.NET Framework), then named it cardValidator.
Step 2: Set up the base framework. Under the cardValidator solution, create the Services and Models folder, and associated .cs files under each. Program.cs is created in the root folder. The Models folder holds the request and response objects (ccValidateRequest.cs and ccValidateResponse.cs) that are used when talking to the API and the Services folder stores the code (ccValidateService.cs) that sends the credit card number to the API and returns the results.
Step 3: Next, we start up GitHub Copilot Chat. A conversation is then started to develop code that will enable creation of the service, call the API, and return of the results. Here is the prompt I provided: I am going to use an API to connect to Elysia Credit Card Validator with URL for a POST to https://elysiatools.com/en/api/tools/credit-card-validator. I have created the framework which includes Models and Services folders. In Models, I have ccValidateRequest.cs and ccValidateResponse.cs but they are not yet filled in with code. Inside Services, I have ccValidateService.cs that also has no code in it yet. Finally, I also created Program.cs within the root folder. I now need guidance to help me create the content of these three C# pieces of code. Let's start with ccValidateRequest.cs.
Step 4: Now, GitHub Copilot generates the code for ccValidateRequest.cs, ccValidateResponse.cs, ccValidateService.cs, and Program.cs using Elysia's API documentation. It often adds a lot more code than may be necessary or may even skip something that is obviously important.
Step 5: Next, we attempt to build the code that was generated and saved locally to my Visual Studio workspace.
Step 6: This worked and generated working code that connected to the API and validated my sample credit card; however, the human-readable report that was returned showed "Card Type:" as "text." I wanted to see Visa shown here. I tried several times to explain to GitHub Copilot that it was not pulling the card type, but instead a value describing the field content. Ultimately, I gave up and manually went into the code to determine how to fix the issue. This is an important aspect of using AI tooling. AI can't always fix or do everything. You still need to be able to understand what is going on in the code to both fix issues like this and understand when other things go wrong in production.
Step 7: Success! After rebuilding the code with my edits, the output now properly shows the "Card Type:" as Visa. AI tools can be extremely helpful and enable rapid development, but when things don't work as expected, a competent programmer is invaluable.
